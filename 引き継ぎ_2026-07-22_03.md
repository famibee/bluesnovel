## 次セッションでの実装方針（要決定）

### 推奨：A案「本家に合わせた実行時定義」

**理由：**
1. 設計が明確で迷いがない
2. bluesnovel は本家の簡略版だから、実装パターンは本家に準拠すべき
3. 複雑性は避けられないが、設計の正確性がより重要

**実装手順：**

1. **Constructor での抽出は不要** ← 実行時に処理するため
   
2. **メイン switch に追加**
   ```typescript
   case 'macro': {
       // [macro name=xxx] を検出
       const name = args.name ?? '';
       if (!name) throw '[macro] nameは必須です';
       
       // 本家の #macro() 同様、ハンドラを動的登録
       // （具体的な登録先は設計検討が必要 → #hTag 相当？）
       
       // [endmacro] までのトークンをスキップ
       while (this.#idx < this.#aToken.length) {
           const tkn = this.#aToken[this.#idx++];
           const {name: nm} = ScriptEngine.parseTag(tkn);
           if (nm === 'endmacro') break;
       }
       continue;
   }
   
   case 'endmacro':
       throw '[endmacro] 対応する[macro]がありません';
   ```

3. **マクロ呼び出し**
   - マクロ名をタグ名として直接呼び出し（本家同様）
   - または `[%macroname]` 形式（簡略版独自？）
   - 設計検討が必要

4. **mp: 名前空間の管理**
   - `VarStore` に `mp` を追加（済み）
   - マクロ実行時に `mp:` に引数を設定
   - `[return]` で復元
