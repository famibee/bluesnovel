# bluesnovel 新規エンジン開発 引き継ぎ（1回目：プラン検討＋試作実装）

日付: 2026-07-19
対象リポジトリ: `bluesnovel`（`skynovel_esm` を参考にした完全新規エンジン）

## 0. 前提・参考にした資料

- 参考実装: `skynovel_esm/src/sn/` 一式（PixiJS + 独自シナリオ解析エンジン）
- 移植先: `bluesnovel/src/sn/`（既に `AnalyzeTagArg.ts` `Areas.ts` `CallStack.ts` `CmnInterface.ts`
  `CmnLib.ts` `Config.ts` `ConfigBase.ts` `Grammar.ts` `SysBase.ts` は移植済み）
- 表示層: PixiJS → React（`bluesnovel/src/components/*.tsx`）へ刷新。すでに
  `Main.tsx` `Stage.tsx` `TxtLayer.tsx` `GrpLayer.tsx` `store/store.tsx` `ts/Memento.ts`
  の骨組みが作りかけで存在しており、本セッションではこれを土台として活用した。

## 1. 全体構造（プラン）

### 1.1 基本処理ループ
`skynovel_esm/src/sn/Main.ts` の `#main()` がシナリオ解析の基本形。
トークンを1つずつ取り出し、先頭文字コードで分岐する。

> 参考原文（skynovel_esm/src/sn/Main.ts:226-229）:
> `// [ タグ開始` の後、`if (uc === 91) { ... if (this.#scrItr.isBreak(token)) return; ...`

> 参考原文（skynovel_esm/src/sn/Main.ts:264-266）:
> `// 文字表示` としてタグでもコメントでも変数操作でもないトークンをそのまま文字表示に回している。

bluesnovel でもこの「トークン列を先頭から順に処理し、[l][p][s] 等で止める」という
基本設計を踏襲する。ただし PixiJS 依存除去・React化に伴い、表示処理は
「状態（ストア）を更新する」だけに責務を絞り、実際のDOM描画はReactに委譲する。

### 1.2 タグ辞書（T_HTag）
`bluesnovel/src/sn/Grammar.ts` に全タグの型定義は既に移植済み（skynovel_esm と同一の
タグ名・属性を維持）。[l][p][s] の定義は下記の通り。

> 参考原文（bluesnovel/src/sn/Grammar.ts:248-250）:
> `l : TTag; // 行末クリック待ち` / `p : TTag; // 改ページクリック待ち` / `s : TTag; // 停止する`

本実装ではこの3タグを「状態確定・停止ポイント」として扱う設計は変更しない。
（将来的に全タグを実装する際も、この3タグの意味論はそのまま流用できる）

### 1.3 表示処理（3レイヤ、試作範囲）
- 文字レイヤ：`TxtLayer.tsx`。試作は瞬時表示（1文字ずつのアニメは後回し＝指示通り）
- 画像レイヤ：`GrpLayer.tsx`。試作は画像1枚のみ（face合成・グループ位置指定は後回し）
- 動画・音声：試作では完全省略

### 1.4 読み戻り（PageUp/PageDown）とReactの利点
既存の `ts/Memento.ts` の `Caretaker` クラスが「キー（`fn:idx`）ごとに複数コンポーネントの
Memento（状態スナップショット）を記録し、`prevKey()`/`nextKey()` で状態を復元する」設計を
既に持っていた（試作前から実装済み）。`Stage.tsx` は既に

```tsx
sys.caretaker.update(()=> new Memento(JSON.stringify(aLay)));
```

の形で、画像・文字レイヤの状態（`aLay`）をまるごとJSON化してMementoに保存している。
これはReactの「状態(state)を丸ごと差し替えれば勝手に再描画される」特性をそのまま
活かせる設計であり、pixi.js版で必要だった「差分だけ手動で戻す」実装より大幅に簡潔になる。
→ 今回の試作では、この既存の仕組みに対して
**シナリオエンジン側が停止ポイントごとに `caretaker.push(key)` を呼ぶ**
処理を追加しただけで、PageUp/PageDownによる読み戻り・読み進めが機能するようにした。

## 2. 今回実施した実装（試作）

### 2.1 新規: `src/ts/ScriptEngine.ts`
- `.sn` 風スクリプトの超簡略トークナイザ＋実行器。DOM/fetch非依存の純粋なクラスにして、
  `bun test` で直接検証できるようにした（既存の `test/*.test.ts` と同じ `it`/`expect` 形式）。
- 対応タグ（試作範囲。属性名は本家 `Grammar.ts` の TArg 定義に合わせている）:
  - `[add_lay layer=NAME class=GRP|TXT]` … レイヤ追加
  - `[current layer=NAME]` … デフォルト文字レイヤ切替
  - `[lay layer=NAME pic=FILENAME]` … 画像レイヤの絵を変更（簡略：pic属性のみ）
  - `[r]` … 改行、`[er]` … 現在の文字レイヤをクリア
  - `[jump label=*label]` … 同一ファイル内ラベルへジャンプ（他ファイルへのjump/call、
    マクロ、if分岐、しおり等は未対応 = 次回以降）
  - `[l]` `[p]` `[s]` … 停止ポイント。`{t:'stop', kind, key}` アクションを返し、
    `key = "${fn}:${idx}"` を `ScriptMng` が `sys.caretaker.push(key)` に渡す
  - それ以外のタグはすべて無視（例外を投げない。試作を止めないため）
- 地の文（プレーンテキスト）はトークナイザが「`[`でも改行でもない連続文字」を
  1トークンとして扱うため、1トークン＝1回の表示更新（＝瞬時表示）になる。
  1文字ずつのアニメーションは指示通り完全に後回し。

### 2.2 更新: `src/ts/ScriptMng.ts`
- ダミーのジェネレータ実装を撤去し、`ScriptEngine` を使って実際にスクリプトを解析・
  実行するように変更。
- `load(fn)`: `sys.cfg.searchPath(fn, SEARCH_PATH_ARG_EXT.SCRIPT)` → `fetch` でスクリプト
  本文を取得し `ScriptEngine` を生成。**アセット一式（`path.json` 等）が未整備でも
  試作の動作確認ができるよう**、fetch失敗時は内蔵のサンプルシナリオ `SAMPLE_SN`
  にフォールバックする（本実装ではこのフォールバックは撤去し、正しくロードエラーとして
  扱うこと＝下記TODO参照）。
- `go()`: `ScriptEngine#step()` を呼び、返ってきた `T_ENGINE_ACTION[]` を
  ストアの `addLayer`/`chgPic`/`chgStr` 呼び出しに変換する。`stop` アクションが来たら
  `sys.caretaker.push(act.key)` を呼び、その時点の表示状態を読み戻り対象として記録する。

### 2.3 更新: `src/store/store.tsx`
- `isReadBack: boolean` と `setReadBack` を追加。読み戻り中（`Caretaker.isLast()===false`）
  かどうかをUI側（`TxtLayer`）に伝えるためのフラグ。

### 2.4 更新: `src/components/Main.tsx`
- キー割り当てを指示通りに変更:
  - **Space / クリック**: `next()`。読み戻り中なら履歴を1つ進める、最新位置なら
    シナリオを進める（`scrMng.go()`）
  - **PageDown**: `next()` と同じ扱い（読み進め）
  - **PageUp**: `prev()`。履歴を1つ戻す（`sys.caretaker.prevKey()`）
  - 旧実装の ArrowUp/ArrowDown 割り当ては置き換えた
- `next()`/`prev()` の中で `setReadBack(! sys.caretaker.isLast())` を呼び、読み戻り状態を
  ストアに反映する。

### 2.5 更新: `src/components/TxtLayer.tsx`
- `isReadBack` を参照し、読み戻り中は文字色を黄色にする（指示通り）。
- ついでに `white-space: pre-wrap` を追加（`[r]` による改行を画面に反映するため。
  無いと `\n` が見た目上無視されてしまう）。

### 2.6 更新: `src/components/GrpLayer.tsx`
- 試作の安全策として、`sys.cfg.searchPath()` が例外を投げても画面ごと落ちないよう
  `try/catch` でラップした（`path.json` 等のアセット一式が未整備な開発初期でも、
  文字レイヤ側の動作確認だけは継続できるようにするため）。
  **本実装ではこの握りつぶしは撤去し、正しくロードエラー表示に戻すこと。**

### 2.7 新規テスト: `test/ScriptEngine.test.ts`
- トークナイザ・タグ属性パーサ・停止ポイントでの `step()` 分割・ラベルジャンプ・
  未対応タグの無視・コメント無視を検証する単体テスト。DOM非依存なので高速。

## 3. 試作の既知の割り切り（次回以降で本実装すべき事項）

指示にある通り「試作ではダミー・超簡略処理で良い」の方針に沿って、以下は意図的に
簡略化・省略した。本実装フェーズで `skynovel_esm/src/sn/ScriptIterator.ts` を参照しながら
拡充すること。

1. **1文字ずつの文字送り演出**（`autowc`、`ch_in_style` 等）は完全未対応。現状は
   トークン単位の瞬時表示のみ。
2. **画像の複数枚重ね合わせ（face合成）・グループ位置指定/移動**は未対応。
   `GrpLayer.tsx` は画像1枚固定。
3. **動画・音声**（`playbgm` `playse` `wv` 等）は全タグ無視扱いで何もしない。
4. **if/elsif/else/endif の条件分岐、macro、call/return（コールスタック）**は未対応。
   `bluesnovel/src/sn/CallStack.ts` は型だけ移植済みなので、実装時はこれを使う。
   参考: `skynovel_esm/src/sn/ScriptIterator.ts` の `#if()` `#call()` `#return()` 一帯。
5. **他ファイルへの jump/call、しおり（save/load）**は未対応。`ScriptEngine` は
   1ファイル分のトークン列のみを保持する設計なので、複数ファイルをまたぐ制御は
   `ScriptMng` 側でファイルごとに `ScriptEngine` インスタンスを切り替える形で
   実装する必要がある（`#hEngine` に複数保持する土台だけは用意済み）。
6. **アセット未整備時のフォールバック**（`SAMPLE_SN`、`GrpLayer` の `try/catch`）は
   試作限定の仮対応。本実装では `path.json` 等のアセット定義を用意した上でこれらを撤去し、
   正規のロードエラー処理（`ScriptMng.myTrace(..., 'ET')` 等）に統一すること。
7. **既読/未読管理（`Areas.ts` 活用）、デバッガ連携、スキップ・オート**は未着手。
8. **`[l]`と`[p]`の意味の違い**（行末クリック待ちか改ページ待ちか）を、今回は
   Caretakerへの記録処理としては区別していない（どちらも同じ`stop`扱い）。
   本実装では `[p]` 時にページ送り効果（文字クリア等）を入れる想定。
